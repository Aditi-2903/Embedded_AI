# Chapter 11 — Selecting Models for Constrained Deployment

> *The student learns to evaluate and compare model architectures for embedded deployment, selecting on constraint profile rather than benchmark accuracy alone.*

**Part III — Model-Aware Design**

---

## Learning Outcomes

- **LO11.1** [Analyze] Compare model architectures on memory, compute, power, and accuracy tradeoffs for a specified deployment target
- **LO11.2** [Evaluate] Select a model architecture appropriate for a constrained embedded target and justify the selection against the constraint profile
- **LO11.3** [Apply] Use deployment-aware benchmarks (latency on target hardware, not accuracy on ImageNet) to evaluate model candidates

---

> **The transition this chapter makes:** Until now, the model was given. From here, it is chosen. This is the shift from inference-first to model-aware thinking.
